Notes

  • Lighthouse score improved from 85 → 86. FCP −100ms, TBT −20ms, CLS zeroed — positive results from commit Remove lazy load from image to improve LCP.
  • Trace LCP on /: the LCP element (Blastoise-Mega.webp) is discovered via JS chunk with 496ms of load delay, followed by 953ms of render delay. The 325ms ForcedReflow in _updateDimensions (chunk-4HAY4XJN.js) is the main cause of the render delay. Under Lighthouse throttling LCP held at 2.5s — real user impact is neutral.
  • /team-vs-many and /many-vs-team have elevated LCP (1,337ms and 1,241ms) with render delay dominating (99% of LCP). The LCP element is text (div.damage-result), so there is no image to optimize — the bottleneck is initialization JS. DOM exceeds 1,600 elements on these routes.
  • ForcedReflow on every VGC route comes consistently from _updateDimensions in chunk-4HAY4XJN.js (Angular Material component). Worth investigating whether it is the probability-calc mat-slider being initialized on all routes via shared module.
  • ForcedReflow on /type-calc dropped from 318ms → 146ms — significant improvement.
  • Smogon (jQuery 1.9.1): 613ms ForcedReflow, 11,139 DOM elements, 2,470ms Lighthouse TBT.
  • NCP: 0.25 CLS in lab trace, 6,565 DOM elements, 1,030ms TBT.
  • /how-to-use is the fastest route: 586ms LCP, 0ms ForcedReflow — reference baseline for the others.
  • ImageDelivery on /how-to-use flagged 3.3MB of images — page screenshots could be converted to WebP.
  • TBT per route (new, unthrottled trace): routes with pokemon-build land between 230–369ms (Needs improvement), /how-to-use at 95ms (Good). Not directly comparable to Lighthouse TBT (80ms throttled mobile) — the two metrics measure different windows. Per-route bottleneck is consistent: 2–4 long tasks during boot, dominated by initial bundle + slider render.
  • INP (new, via simulated EV slider interaction): 80ms (Good) across 83 collected events. Breakdown: input delay 33ms, processing 0ms, presentation 47ms. Confirms JS is not the bottleneck — the cost is in layout/paint of the Material slider. Target is still mdc-slider__input because P4-C was reverted (misaligned pipes). Even so, INP is well below the 200ms threshold.
