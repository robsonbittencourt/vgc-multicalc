Notes

  • Lighthouse score delta is substantial: production scores 57 vs local 86 (+29 points). The primary driver is LCP/FCP: production registers 9.1s/8.7s under mobile throttling vs 2.5s/1.9s for the local build. This gap is expected — the local build includes all SEO-branch optimizations (webp images, preload hints, etc.) not yet deployed to production.
  • The Lighthouse throttled LCP difference (9.1s prod vs 2.5s local) should not be confused with the unthrottled trace (1,582ms prod vs 1,652ms local). Throttled Lighthouse simulates a slow mobile network; the trace runs on the local machine with no network throttling — they measure different conditions. The production trace LCP of 1,582ms is measured from a local machine loading the live site, so it reflects developer network speed, not real-user conditions.
  • CrUX field LCP for production (1,961ms p75) is far better than the Lighthouse throttled result (9.1s) — real users are on faster connections. The throttled Lighthouse result is a stress-test, not a typical-user scenario.
  • Production TBT is 0ms in Lighthouse, while local build shows 60ms. Both are within the "Good" threshold (<200ms). The 60ms local TBT likely comes from the additional Angular initialization overhead introduced by new features in the SEO branch. Not a concern.
  • Trace load delay is dramatically different: production shows 1,341ms load delay (image download from CDN), while local shows only 24ms (localhost). This is expected — local assets are served instantly, while production assets traverse the network. Render delay is the inverse: local spends 1,605ms in render (Angular initialization dominates after fast asset load), while production spends only 238ms in render after the long load delay.
  • ForcedReflow is 395ms in production vs 322ms in local build. Both originate from the same _updateDimensions Angular Material call (mat-slider). The local build's improvement (−73ms) is consistent with the webp image conversion reducing layout recalculation scope.
  • Production INP (CrUX p75: 205ms) crosses the Needs Improvement threshold. The local lab measurement (155ms) stays in Good. This is worth tracking post-deploy — the CrUX value aggregates all real interactions, while the lab value measures only the EV slider keydown specifically.
  • DOM size is virtually identical (1,751 prod vs 1,756 local) — the SEO-branch changes did not add meaningful DOM nodes.
