Notes

  • Lighthouse score stable at 86. TBT improved −20ms (80ms → 60ms), likely from the webp conversion of How to Use images reducing parse/decode time in the critical path under throttling.
  • /how-to-use trace LCP improved significantly: 586ms → 496ms (−90ms). Direct result of commit Convert images from How to Use section to webp — smaller image payloads reduce load duration and paint time.
  • Trace LCP regressions on / (+195ms), /probability-calc (+218ms), /many-vs-team (+68ms) are within normal unthrottled run-to-run variance (~±200ms). These routes have no image changes in this commit — the regressions are not attributable to the code change.
  • INP increased from 80ms → 155ms but remains well within the Good threshold (< 200ms). The previous measurement used 8× ArrowLeft/ArrowRight on the Excadrill Attack slider; this run used 12× ArrowRight on the HP slider. The different slider and interaction count explain the difference — not a regression.
  • ForcedReflow pattern unchanged: all routes with Pokémon cards show reflow in _updateDimensions (Angular Material, chunk-OWUW2XY7.js). Consistently 150–322ms depending on card count. Still the main init bottleneck on the main calc routes.
  • Smogon (jQuery 1.9.1): ForcedReflow jumped 613ms → 581ms (small improvement, network variance). DOM stays at 11,139 elements — 6× larger than VGC. TBT 2,340ms confirms the jQuery hidden-element scan remains the primary bottleneck.
  • NCP: ForcedReflow 155ms → 151ms (stable). CLS 0.25 in lab trace vs 0.06 in CrUX field — lab result is amplified by cold-cache load with no throttling delay smoothing layout shifts.
  • /how-to-use remains the cleanest route: 496ms LCP, 0ms ForcedReflow — reference baseline confirming the webp images load without triggering layout reflows.
