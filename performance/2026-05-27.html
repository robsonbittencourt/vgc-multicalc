Notes

  • Score improved +10 vs previous prod (57 → 67). LCP dropped from 9.1s → 1.4s and FCP from 8.7s → 1.0s — the SEO branch optimizations (webp images, preload hints) are now live in production and delivering the expected gains. However, TBT regressed from 0ms → 1,280ms under mobile throttling, which is the main reason the score did not rise further. The commit feat: Add Team List generator (57a33d0e) is the most likely source of this TBT regression — the new feature likely adds synchronous initialization work in the critical path.
  • INP has been in "Needs improvement" since the first prod measurement. CrUX p75 was 205ms on 2026-05-21 and is now 213ms (+8ms) — a marginal increase within noise. Lab INP measured directly on prod today: 207ms. Processing duration (158ms) is the dominant phase, consistent with Angular signal recalculation on every keydown event across the card tree.
  • CrUX field LCP is healthy at 2,010ms (Good). Real users experience a good LCP despite the throttled Lighthouse showing 1.4s — both are in the Good range. The 2026-05-21 field LCP was 1,961ms (stable, no regression in real user experience).
  • ForcedReflow source unchanged across all VGC routes: _updateDimensions in chunk-UU4EPT6I.js (Angular Material mat-slider). Values range from 33ms (/how-to-use) to 291ms (/). The home route reflow decreased slightly (322ms → 291ms), likely from different initial card state in prod vs local.
  • Competitors stable. Smogon score 71 (unchanged). NCP score 51 (unchanged). Smogon ForcedReflow: 556ms (jQuery 1.9.1 hidden-element scan, DOM 11,139 elements). NCP ForcedReflow: 139ms (jQuery 3.1.1, DOM 6,565). VGC's Lighthouse LCP (1.4s) now outperforms Smogon (1.6s) but TBT (1,280ms) remains below Smogon's (2,100ms) — VGC is faster to paint but more blocking during load.
  • Trace LCP deltas vs local are within noise for most routes. The +364ms on / is above the ±200ms run-to-run variance threshold and likely reflects real network latency from the prod CDN vs localhost (load delay: 251ms in prod vs 24ms local). Routes without load delay show render-dominated LCP, which is consistent with local measurements.
  • /how-to-use remains the fastest route: 483ms LCP, 33ms ForcedReflow — confirming webp image conversion is live in production and delivering the expected improvement.
